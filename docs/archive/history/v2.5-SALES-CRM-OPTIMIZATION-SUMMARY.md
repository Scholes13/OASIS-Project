### 2. **Eager Loading Optimization**

**Before (N+1 Problem):**
```php
Activity::with(['user', 'contact'])->paginate(20);
// Result: 1 + 20 + 20 = 41 queries ❌
```

**After (Optimized):**
```php
Activity::with([
    'user:id,name,email',  // Select only needed columns
    'contact:id,code,name,company,email,phone'
])->paginate(20);
// Result: 1 + 1 + 1 = 3 queries ✅
```

### 3. **FULLTEXT Search** (10-50x Faster)

**Before (LIKE Search - Slow):**
```php
->where('title', 'like', "%{$search}%")
->orWhere('description', 'like', "%{$search}%")
// Full table scan: 200-500ms on 10k+ rows ❌
```

**After (FULLTEXT - Fast):**
```php
->whereRaw(
    "MATCH(title, description) AGAINST(? IN BOOLEAN MODE)",
    [$search.'*']
)
// Index scan: 20-50ms on 100k+ rows ✅
```

**Supports:**
- `john`: Exact word
- `john*`: Prefix match (johnathan, johnny)
- `+john +doe`: Must contain both
- `john -doe`: Contains "john" but not "doe"

### 4. **Stats Caching** (1 Query + Cache)

**Before (4 Queries):**
```php
$total = Activity::count();      // Query 1
$completed = Activity::where('status', 'completed')->count();  // Query 2
$planned = Activity::where('status', 'planned')->count();      // Query 3
$today = Activity::whereDate('activity_date', today())->count(); // Query 4
// Result: 4 queries every page load ❌
```

**After (1 Query + Cache):**
```php
Cache::remember("activity_stats_{$buId}_{$userId}", 300, function () {
    return Activity::selectRaw('
        COUNT(*) as total,
        SUM(CASE WHEN status = "completed" THEN 1 ELSE 0 END) as completed,
        SUM(CASE WHEN status = "planned" THEN 1 ELSE 0 END) as planned,
        SUM(CASE WHEN DATE(activity_date) = CURDATE() THEN 1 ELSE 0 END) as today
    ')->first();
});
// Result: 1 query on cache miss, 0 queries on cache hit ✅
```

**Cache Strategy:**
- **TTL:** 5 minutes (300 seconds)
- **Key Pattern:** `{module}_stats_{bu_id}_{user_id}`
- **Invalidation:** Clear on create/update/delete

### 5. **Cache Invalidation**

**File:** `app/Livewire/Modules/SalesCrm/ActivityForm.php`

```php
protected function clearModuleCaches(?Contact $contact): void
{
    $buId = session('current_business_unit_id');
    $userId = Auth::id();

    // Clear activity stats cache
    Cache::forget("activity_stats_{$buId}_{$userId}");

    // Clear contact stats cache if contact was created/updated
    if ($contact) {
        Cache::forget("contact_stats_{$buId}_{$userId}");
        
        // Also clear assigned user's cache if different
        if ($contact->assigned_to && $contact->assigned_to != $userId) {
            Cache::forget("contact_stats_{$buId}_{$contact->assigned_to}");
        }
    }
}
```

**Why Multi-User Cache Clear?**
- Admin creates activity → clears admin's cache
- Contact assigned to sales user → clears sales user's cache
- Both see fresh data immediately

## Troubleshooting

### Issue 1: "Unknown column in field list"

**Symptom:** Error when visiting activities/contacts page

**Cause:** Eager loading selecting non-existent columns

**Solution:**
```php
// Check model relationships
Activity::with(['user:id,name'])->first();
// If error, column 'name' might not exist in users table

// Fix: Check users table schema
DB::select("DESCRIBE users");
```

### Issue 2: FULLTEXT Search Not Working

**Symptom:** Search still slow

**Solution:**
```sql
-- Check if FULLTEXT index exists
SHOW INDEX FROM activities WHERE Key_name = 'idx_activities_search';

-- If not exists, run migration again
php artisan migrate:rollback --step=1
php artisan migrate

-- Test manually
ALTER TABLE activities ADD FULLTEXT INDEX idx_activities_search (title, description);
```

### Issue 3: Cache Not Clearing

**Symptom:** Stats not updating after create

**Solution:**
```php
// Check cache driver
php artisan tinker
Cache::put('test', 'value', 60);
Cache::get('test'); // Should return "value"

// If null, check .env
CACHE_DRIVER=database  // or file, redis
php artisan config:clear
php artisan cache:clear
```

## Best Practices (Quick Reference)

### ✅ DO
1. Always eager load relationships with specific columns
2. Use composite indexes for common WHERE clauses
3. Cache stats calculations (5-10 min TTL)
4. Use FULLTEXT for multi-column text search
5. Clear caches on mutations
6. Test with large datasets (10k+ rows)

### ❌ DON'T
1. Don't select all columns if not needed
2. Don't forget eager loading (causes N+1)
3. Don't use LIKE for large text searches
4. Don't cache forever (use reasonable TTL)
5. Don't forget cache invalidation
6. Don't add indexes blindly
