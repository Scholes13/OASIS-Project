# Archive Index

This directory holds historical documentation that should remain in the repository for reference but should not stay at the top level of `docs/`.

## Structure

- `bug-fixes/`: historical bug investigations and one-off fixes
- `tasks/`: historical task completion reports
- `history/`: older summaries, optimization notes, and project writeups

## Guidance

- Archive docs should remain readable, but they are not the primary source of truth for current architecture or workflow.
- When a top-level doc becomes mostly historical, move it here instead of leaving it mixed with canonical docs.
